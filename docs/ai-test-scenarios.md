# AI-помощник: тестовые сценарии

## Как тестировать
Каждый сценарий: отправить команду в чат → проверить результат на GitHub → записать pass/fail.

---

## 1. Добавление лейбла по названию issue
**Команда:** `добавь лейбл P1-critical к задаче "Заполнить телефон компании" в Beer_bot`
**Ожидание:**
- AI находит issue по названию через list_repo_issues
- Не угадывает номер
- Добавляет лейбл `P1-critical` (не `critical`, не `P1`)
**Проверка:** `gh issue view [номер] --repo Sergio1990-1/Beer_bot --json labels`

## 2. Создание issue с контекстом
**Команда:** `создай issue для Sewing-ERP: добавить валидацию email в форме заказа`
**Ожидание:**
- AI сначала читает docs/ проекта
- Создаёт issue с описанием в формате проекта
- Добавляет в MakeIT Tracker
**Проверка:** issue появился в GitHub + в Project board

## 3. Анализ velocity
**Команда:** `проанализируй скорость разработки Sewing-ERP за последние 2 недели`
**Ожидание:**
- Загружает issues с датами закрытия
- Считает по АКТИВНЫМ дням (не календарным)
- Показывает разбивку по дням
- Предупреждает если данных мало
**Проверка:** цифры совпадают с `gh issue list --state closed --json closedAt`

## 4. Вопрос "сколько сделано сегодня"
**Команда:** `сколько задач закрыто сегодня?`
**Ожидание:**
- Отвечает конкретным числом из контекста
- Перечисляет задачи с названиями
- НЕ говорит "у меня нет данных"
**Проверка:** число совпадает с Done count

## 5. Создание milestone с дедлайном
**Команда:** `создай milestone "Sprint 1" в Beer_bot с дедлайном 15 апреля`
**Ожидание:**
- Создаёт milestone с правильной датой
- Подтверждает создание с URL
**Проверка:** `gh api repos/Sergio1990-1/Beer_bot/milestones --jq '.[].title'`

## 6. Несуществующий issue (anti-hallucination)
**Команда:** `добавь лейбл bug к issue #99999 в Beer_bot`
**Ожидание:**
- AI НЕ говорит "готово"
- Показывает ошибку от GitHub API
**Проверка:** issue не существует

## 7. Статус-апдейт для клиента
**Команда:** `напиши статус-апдейт для клиента mankassa-app`
**Ожидание:**
- Использует реальные данные (прогресс, milestones, закрытые issues)
- Текст подходит для отправки клиенту
- Цифры совпадают с дашбордом

## 8. Финансовый вопрос
**Команда:** `кому нужно выставить счёт?`
**Ожидание:**
- Показывает проекты где остаток > 0
- Учитывает процент выполнения
- Не рекомендует счёт при 0% прогресса

## 9. Привязка issues к milestone
**Команда:** `привяжи все P1 задачи Beer_bot к milestone Sprint 1`
**Ожидание:**
- Находит P1 issues через list_repo_issues
- Находит milestone через list_milestones
- Привязывает каждый issue
**Проверка:** issues появились в milestone на GitHub

## 10. Ошибка при невалидном лейбле
**Команда:** `добавь лейбл "несуществующий" к первой задаче Sewing-ERP`
**Ожидание:**
- AI создаёт лейбл если его нет (текущее поведение)
- ИЛИ предупреждает что лейбла нет
- НЕ молча игнорирует
